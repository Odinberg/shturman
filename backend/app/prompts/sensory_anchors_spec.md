# Сенсорные якоря: техническая спецификация

## Обзор
Направление работает на стыке телесных ощущений и предиктивной аналитики. 
Капитан 3 раза в день касается кнопки в приложении и выбирает 1 из 5 телесных ощущений. 
Без текста — только тело.

## Пользовательский поток

### 1. Чекин ощущений (3 раза в день)
```
[Push-уведомление] «Что чувствует тело?»
  → Капитан открывает приложение
  → 5 кнопок-ощущений:
      🔴 Напряжение
      🟢 Лёгкость
      🔵 Холод
      🟠 Жар
      ⚪ Онемение
  → Одно касание — done. Экран гаснет.
```

### 2. Сбор контекстных данных (пассивно, с разрешения)
- **Акселерометр:** как капитан держит телефон (угол наклона, тремор, сила хвата — через API датчиков)
- **Время касания:** таймстемп + день недели
- **Геолокация (опционально):** место чекина (дом/работа/транспорт)

### 3. Аналитика (на стороне бэкенда)

**Телесный компас (бонус за 14 дней чекинов):**
- Модель: корреляция «телесный паттерн → эмоциональный чекин через 2 часа»
- Вход: 14+ дней сенсорных чекинов + эмоциональные чекины
- Выход: предиктивная модель «если сейчас X → через 2 часа вероятно Y»
- Реализация: простой байесовский классификатор или логистическая регрессия на временных рядах
- Push: «Через час может накатить тревога. Включить ресурсную музыку на 5 минут раньше?»

**Кинетический дневник (бонус за 7 дней):**
- Генерация: Canvas/SVG-анимация движущихся линий на основе 21 точки (7 дней × 3 чекина)
- Алгоритм: каждая точка → вектор (цвет = ощущение, длина = время между чекинами, угол = время суток)
- Вывод: GIF-анимация «танца настроения» за неделю

**Якорь спокойствия (бонус за момент «спокойствие» в дневнике):**
- Триггер: капитан пишет слово «спокойствие» в дневнике
- Система запоминает: тактильный паттерн в момент написания
- Воспроизведение: вибро-паттерн на часах/телефоне, повторяющий ритм дыхания капитана в тот момент
- Технически: запись частоты дыхания (опционально через камеру/микрофон) или использование среднего ритма 4-7-8

## Технические требования

### Mobile SDK
- **iOS:** CoreMotion (акселерометр), HealthKit (дыхание, пульс — опционально), UNUserNotificationCenter (push)
- **Android:** SensorManager (акселерометр), Google Fit (опционально), Firebase Cloud Messaging (push)
- **Web:** DeviceMotionEvent API, Vibration API, Notification API

### Бэкенд
- **Хранение:** временные ряды в PostgreSQL (timescale или партиции по дате)
- **Модель предсказания:** Python scikit-learn, простая логистическая регрессия
- **Вибрация:** Web Vibration API pattern: `navigator.vibrate([длительность, пауза, ...])`
- **Push:** Firebase Cloud Messaging + APNs

### Endpoints
```
POST   /api/sensory/checkin      — запись чекина
GET    /api/sensory/history       — история чекинов
GET    /api/sensory/predict       — предсказание на сегодня
GET    /api/sensory/kinetic       — данные для танца
POST   /api/sensory/anchor/save   — сохранить якорь
POST   /api/sensory/anchor/play   — воспроизвести якорь
```

## Приоритет реализации
1. **MVP:** чекины (3 кнопки) + хранение + визуализация графика
2. **v1:** Телесный компас (предиктивная модель)
3. **v1.5:** Кинетический дневник (анимация)
4. **v2:** Якорь спокойствия (вибро-паттерн)
5. **v2.5:** Интеграция с носимыми устройствами (Apple Watch, Wear OS)
